VMSCP 介绍

VMSCP 简单介绍

  • VMSCP ｜ 点击跳转官网

VMSCP 是一个服务商代开服务器面板，支持

  • Azure
  • Aws
  • Linode
  • Vultr
  • Do
  • Oracle
  • Scloud

我们有一个疑问，就是为什么需要 VMSCP 这个平台，主要是有如下的理由

  • 像 AWS 等平台都有严格的风控，如果你单 IP 登录大量账号的话，就会导致账号被封，而 VMSCP 采用的是 API 创建服务器，不需要登录账号，所以，减小了封号的概率
  • 如果不是用 VMSCP 自己想要管理 AWS 账号的话，可能需要自己做账号隔离，比如单账号，单IP、单环境，比如 是用 AdsPower + IP 形式做到环境隔离，不过，这样和 VMSCP 比起来相对麻烦一点，但是，安全性是比 VMSCP 高的，但是，成本也拉上去了

整体而言，VMSCP 可以帮助你管理多个账号，因为平台是专门做这个的，所以，使用起来没啥难度，很顺手。但是，VMSCP 只是帮你减少封号机率，并不是规避封号。要知道，即便是你正常使用，定期缴费，还是有可能被封号。

风险提示: 大规模使用 VMSCP 开服务器还是有可能会导致封号的

付费

VMSCP 是一款付费的平台，采用 USDT 缴费。关于，USDT 是什么，在这里超出本文档的范围了，具体细节可以咨询相关人士。

VMSCP 是根据服务商解锁付费的，比如，你用 AWS ，只解锁 AWS 就好了。

使用

本文档，以 AWS 举例，这里认为你已经注册好 VMSCP 了，并且开了 AWS 服务。

如果，你通过渠道购买账号，账号应包含下面的信息

  • 邮箱
  • 登录密码
  • key
  • secret

我们只会用到邮箱、key、secret。另外，按照道理来说，登录密码最好修改，但是，如果你没有做好隔离，频繁登录，然后修改密码，反而会导致账号直接被封。所以，这里有一个矛盾点就是，你可能修改不了登录密码，所以，这个时候渠道的人品非常重要。

登录 AWS

选择「AWS 实例管理/账号列表」-> 添加 -> 填入相关的信息

只需要填写

  • 邮箱地址
  • AWS key
  • AWS secret
  • vcpu
    • 这个需要问一下账号信息，通常是 8 或者 32
  • 其他 保持默认就好

然后点击确认。

一切顺利的话，你的账号会出现在界面中。

图中红框处，可以查询账号的可用信息，如果，「检查可用性」这一块显示失败，说明账号已经被 AWS 封了。

创建 AWS EC2

以东京创建为例，选择上面图中的 「创建VM」

  • 有时候我们发现账号中已经有一个服务器被创建了，这是因为，白号通常需要开启一个服务进行测试，只需要把这个实例终止就好了

先选择左侧红框中的区域 -> 添加，然后选择相关实例的各种配置，最后点击确定就好了。

SSH 连接

使用 VMSCP 开启的实例，具备下面特点

  • 用户是 root
  • 默认开启全部端口

可以使用 ssh 工具，将相关参数填进去就可以连接了。

这里有一点需要注意，连接实例必须等 「实例状态」变为 running 才能连接。
